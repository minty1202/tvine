# コミット手順

以下の手順を順番に実行する。

## 1. メモリ更新

- 現在の Issue に対応する `memory/issue-<番号>.md` のチェックリストと設計メモを更新する
- 新たなプロジェクト知見があれば `memory/project.md` も更新する

## 2. CI チェック

- `/usr/bin/make ci` を実行し、lint・テストが通ることを確認する
- 失敗した場合は修正してから再実行する

## 3. コミット

- `git status` と `git diff` で変更内容を確認する
- `git log --oneline -5` で直近のコミットメッセージのスタイルを確認する
- 意味のある単位でコミットを分割する（1 コミットに複数の関心事を混ぜない）
- 変更内容に合ったコミットメッセージを作成する
- `Co-Authored-By` は基本つけない
- HEREDOC 形式でメッセージを渡す
