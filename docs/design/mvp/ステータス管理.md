# ステータス管理

## 概要

各セッションの Claude Code が今どの状態にあるかを検出し、セッション一覧に表示する。
ユーザーはステータスを見てどのセッションに注意を向けるべきか判断する。

## ステータスの種類

| ステータス | 意味 | 表示 |
| --- | --- | --- |
| Busy | Claude Code が作業している | 新規デザイン（別途決定） |
| Evaluating | 出力を検出したが確定前 / Busy から出力が途切れた | ブレイルスピナー（⠋⠙⠹…） |
| Idle | 新しい入力を受け付けられる状態 | ● グレー |

### v1-v3 からの変遷

- ~~Waiting（ユーザーの確認・承認待ち）~~ → v3 で廃止。hook で正確に検出できなかったため
- ~~Busy / Idle の 2 状態~~ → v4 で Evaluating（判断中）を追加。3 状態に拡張

## 検出方法

PTY 出力監視で検出する。hook / Plugin / Unix socket は使わない。

### 過去の方式と変更理由

| 方式 | 版 | 問題 |
| --- | --- | --- |
| Plugin hooks + Unix socket | v1 | hook にギャップ（esc 中断、Permission 拒否で Stop が発火しない）。Waiting との競合が解消不能 |
| PTY 出力監視（2 状態） | v2 | ユーザーのキー入力エコーも Busy と検出 |
| hooks + 沈黙フォールバック（Waiting 廃止） | v3 | AskUserQuestion 後の応答で UserPromptSubmit が発火しない。hook イベントへの依存が高い |
| OSC タイトルパース | — | Claude Code の内部実装への依存が高い（ccmanager 方式） |

### v4: PTY 出力監視 + 3 状態遷移

PTY 出力の有無を監視し、3 状態の遷移で Busy を判定する。autocomplete の debounce と同じ原理。

- PTY 出力は最も低レベルな信号で、hook イベントや内部実装に依存しない
- キー入力エコーによる誤検知は Evaluating（判断中）状態で吸収する
- tmux の `monitor-activity` / `monitor-silence` も同じ PTY 出力ベース

### 状態遷移

```txt
Idle → (PTY 出力) → Evaluating → (出力が続く) → Busy
                       ↑                         ↓
                       ↑←←← (連続出力が途切れる) ←↓
                       ↓
                  (沈黙 2 秒) → Idle
                  (出力再開) → Busy
```

- **Idle → Evaluating**: PTY 出力を検出した瞬間
- **Evaluating → Busy**: 出力が閾値時間を超えて継続（キーエコーとの区別）
- **Busy → Evaluating**: 連続した出力が途切れた
- **Evaluating → Idle**: 沈黙が 2 秒続いた
- **Evaluating → Busy**: Evaluating 中に出力が再開した
- **PTY exit → Idle**: Claude Code がクラッシュした場合

### 不要になったもの（v1 から削除）

- ~~Plugin ファイル生成（plugin.json, hooks.json, notify.sh）~~
- ~~Unix socket リスナー（Rust 側）~~
- ~~`--plugin-dir` と環境変数の注入~~
- ~~`status-changed` Tauri イベント~~

### Claude Code 起動コマンド

```bash
# 初回
claude --session-id <uuid>
# 再接続
claude --resume <uuid>
```

`--plugin-dir` や環境変数は不要。

## 通知経路

フロントエンドが `pty-output` イベント（既存）を監視するだけ。バックエンドの変更は不要。

```text
PTY → pty-output イベント → フロントエンド（Evaluating/Busy/Idle 判定）
```

## tvine 復帰時のステータス復元

不要。tvine 再起動時は全セッション Idle（tvine 終了時に Claude Code も終了するため）。

## 表示

セッション一覧のカードにステータスを表示する。

| ステータス | 表示 | 色 |
| --- | --- | --- |
| Busy | 別途決定 | 別途決定 |
| Evaluating | ブレイルスピナー（⠋⠙⠹…） | 別途決定 |
| Idle | ● | グレー |
