# #23 セッション一覧と削除

## 完了時の状態

- サイドバーにセッション一覧が表示される（ブランチ名・ベースブランチ）
- セッションの削除ボタンから確認モーダルを経て、worktree と session.json が削除される
- 一覧取得時に worktree が存在しない orphan セッションが自動除外・削除される（#20）
- セッションが 0 件のときは案内メッセージを表示

## 方針

### ts-rs: 型の自動生成

#### 方針

- `ts-rs` クレートで Rust の kernel 型から TypeScript 定義を自動生成する
- 手書きの TypeScript 型定義を削除し、生成されたファイルに置き換える

#### 実装方法

- workspace に `ts-rs` を追加、kernel の `dev-dependencies` に設定
- `Session` / `SessionId` に `#[derive(TS)]` を追加、`#[ts(export)]` で出力
- 生成先: `src/generated/` に出力（`.gitignore` には入れず、コミット対象とする）
- `make ts-rs` 等のコマンドで `cargo test -p kernel` を実行して生成
- フロントの `createSession.ts` にある手書きの `Session` 型を削除し、生成ファイルからの import に切り替える

### api: delete_session handler

#### 方針

- worktree 削除 → session.json 削除の順で実行
- worktree がすでに存在しない場合（手動削除済み等）は worktree 削除をスキップし、session.json だけ消す。エラーにはしない（正常終了扱い）
#### 実装方法

- `crates/api/src/handler/session.rs` に `delete_session` を追加
- `git_repository().force_remove_worktree()` → 失敗時、worktree パスが存在しなければ無視して続行
- `session_repository().delete()` で session.json を削除
- テスト: 正常系、worktree が既にない場合、worktree 削除失敗時

### api: list_sessions handler

#### 方針

- 一覧取得と遅延 reconcile（#20）を一体で実装する
- handler 内で list → worktree 存在チェック → orphan 削除 → フィルタ済みリストを返す
- 作成順（created_at 昇順）でソートして返す

#### 実装方法

- `crates/api/src/handler/session.rs` に `list_sessions` を追加
- `session_repository().list()` で全件取得
- `worktree_path.exists()` で存在チェック、orphan は `session_repository().delete()` で削除
- 有効なセッションのみ `created_at` 昇順でソートして返す
- テスト: 正常系、orphan フィルタ、空リスト

### Tauri IPC コマンド

#### 方針

- 既存の `create_session` と同じパターンで追加

#### 実装方法

- `src-tauri/src/routes/session.rs` に `list_sessions` / `delete_session` を追加
- `src-tauri/src/routes/mod.rs` の `generate_handler!` に登録
- `list_sessions`: 引数なし、`Vec<Session>` を返す
- `delete_session`: `session_id: String` を受け取る

### フロントエンド: セッション一覧

#### 方針

- TanStack Query の `useQuery` で一覧を取得
- モックアップに合わせた最小限の表示（ブランチ名、ベースブランチ、削除ボタン）
- ステータス表示は今回のスコープ外（別 Issue）
- 0 件時は案内メッセージを表示

#### 実装方法

##### API 関数

- `src/features/sessions/api/listSessions.ts`: `invoke<Session[]>('list_sessions')`

##### Hook

- `src/features/sessions/hooks/useListSessions.ts`: `useQuery({ queryKey: ['sessions'], queryFn: listSessions })`

##### コンポーネント

- `src/features/sessions/components/SessionSidebar.tsx`: `useListSessions` で取得した一覧を表示
- `src/features/sessions/components/SessionCard.tsx`: 各セッションのカード（ブランチ名、base、削除ボタン）
- セッション作成成功時に `queryClient.invalidateQueries({ queryKey: ['sessions'] })` で一覧を更新

### フロントエンド: 削除

#### 方針

- 削除ボタンから確認モーダルを表示し、確定で削除を実行
- 作成モーダルと同じく useMutation でローディング・エラー管理
- 削除成功時に一覧を invalidate

#### 実装方法

##### API 関数

- `src/features/sessions/api/deleteSession.ts`: `invoke('delete_session', { sessionId })`

##### Hook

- `src/features/sessions/hooks/useDeleteSession.ts`: `useMutation` + `onSuccess` で `invalidateQueries(['sessions'])`

##### コンポーネント

- `src/features/sessions/components/DeleteSessionModal.tsx`: 確認モーダル（ブランチ名表示、削除ボタン、キャンセル）
- ボタンと確認モーダルは 1 コンポーネントにまとめる（ユーザーの好み）
