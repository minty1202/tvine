## 方針

### デフォルトブランチの保存

#### 方針

- `initialize_project` のタイミングで、リポジトリのデフォルトブランチを検出して `project.json` に保存する
- セッション作成モーダルのベースブランチ選択で、デフォルト値として使う

#### 実装方法

- client/git に `default_branch()` を追加。`git symbolic-ref refs/remotes/origin/HEAD` で検出し、取れなければ `main` → `master` の順でフォールバック
- kernel の `GitRepository` trait にメソッドを追加
- data の `project.json` に `default_branch` フィールドを追加
- `initialize_project` で呼び出して保存

### Tauri state と invoke コマンドの配管

#### 方針

- setup フックで作成した `AppRegistry` を `manage()` で Tauri state に保持する
- コマンドハンドラーは `State<AppRegistry>` を受け取る形にし、配管は本番と同じ構造にする
- セッション作成コマンドの実処理はスタブ（受け取ってログ出力 + 成功を返す）

#### 実装方法

- setup フックで `app.manage(app_registry)` を追加
- `#[tauri::command]` でセッション作成コマンドを定義（`base_branch`, `branch_name` を受け取る）
- `invoke_handler` に登録
- 実処理は後続 Issue で埋める

### セッション作成モーダル UI

#### 方針

- `src/features/sessions/components/` に配置する（Bulletproof React のドメイン単位の feature 構成）
- Mantine の Modal + フォームコンポーネントを使う
- ブランチ一覧の取得は別 Issue（#17）のため、ベースブランチは project.json のデフォルトブランチを初期値としたテキスト入力にする
- ボタンとモーダルは 1 コンポーネント（`CreateSessionModal`）にまとめる。Layout は配置するだけ

#### 実装方法

- `CreateSessionModal` コンポーネント:
  - `+ New` ボタン + モーダルを内包
  - 開閉は `useDisclosure` で管理
  - 入力項目: ベースブランチ（TextInput）、ブランチ名（TextInput）
  - 送信時に Tauri invoke でバックエンドに送る（現時点は console.log）
- テーマで Button のデフォルト variant を `light` に設定

### フロントエンド側バリデーション

#### 方針

- 送信前にフロントエンドでバリデーションし、不正な入力がバックエンドに届かないようにする
- git の命名規則に準拠したチェックを行う

#### 実装方法

- ブランチ名が空でないこと
- git の命名規則違反のチェック:
  - スペースを含まない
  - `..` を含まない
  - `~`, `^`, `:`, `\` を含まない
  - `/` で始まらない、`/` で終わらない、`//` を含まない
  - `.lock` で終わらない
  - `.` で始まらない
- バリデーションロジックはテスト対象とする
