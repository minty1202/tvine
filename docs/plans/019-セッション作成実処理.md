# セッション作成の実処理（worktree 作成 + session.json 保存）

Issue: #19（親: #4）

## 完了時の状態

- モーダルからブランチ名を入力すると、リポジトリの親ディレクトリに worktree が作成される（例: リポジトリが `~/Development/my/tvine` でブランチ名が `feature/1` → `~/Development/my/tvine-feature-1`）
- `~/.tvine/projects/<project-id>/sessions/<uuid>/session.json` にセッション情報が保存される
- フロントエンドに成功/失敗が返る。途中で失敗した場合は作成済みのリソース（worktree 等）をロールバックし、中途半端な状態を残さない

## 方針

### kernel: Session モデルと trait 定義

- `kernel/src/model/session.rs` に Session 構造体を定義
  - `id: String`（UUID）
  - `branch_name: String`
  - `base_branch: String`
  - `worktree_path: PathBuf`
  - `created_at: String`（ISO 8601）
  - `slot_id` は今回スコープ外なので含めない
- `kernel/src/repository/session.rs` に `SessionRepository` trait を定義
  - `create(&self, session: &Session) -> AppResult<()>` — session.json を保存
- `GitRepository` trait に `create_worktree` を追加
  - `create_worktree(&self, base_branch: &str, branch_name: &str, path: &Path) -> AppResult<()>`

### client/git: worktree 作成

- `git worktree add -b <branch_name> <path> <base_branch>` を `std::process::Command` で実行
- `Client` trait に `create_worktree` を追加
- エラー時は git のエラーメッセージをそのまま返す

### data: session.json の保存

- `data/src/manage/session.rs` に `save_session` 関数を追加
- 保存先: `<storage_dir>/sessions/<session-id>/session.json`
- セッションディレクトリの作成も行う

### adapter: SessionRepositoryImpl

- `adapter/src/repository/session.rs` に実装
- `data::manage::session::save_session` に委譲

### api: create_session ロジック

- `api/src/handler/session.rs` に `create_session` 関数を追加
- 処理の流れ:
  1. UUID を生成
  2. worktree パスを算出（後述）
  3. `GitRepository::create_worktree` で worktree 作成
  4. Session モデルを組み立て
  5. `SessionRepository::create` で session.json を保存
  6. 成功を返す
- ロールバック: ステップ 5 が失敗した場合、ステップ 3 で作成した worktree を `git worktree remove` で削除して元の状態に戻す
- worktree パスの算出: リポジトリの親ディレクトリに `<リポジトリ名>-<ブランチ名>` で配置。ブランチ名の `/` は `-` に置換。例: `~/dev/tvine` + `feature/1` → `~/dev/tvine-feature-1`。ディレクトリ名とブランチ名は異なる（worktree は `tvine-feature-1`、ブランチは `feature/1`）

### registry: AppRegistry に SessionRepository を追加

- `AppRegistryImpl` に `session_repository` フィールドを追加
- `AppRegistry` trait に `session_repository()` メソッドを追加

### src-tauri/routes: スタブを実処理に差し替え

- `routes/session.rs` で `api::handler::session::create_session` を呼び出す
- エラーは `AppError` をそのままフロントに返す
