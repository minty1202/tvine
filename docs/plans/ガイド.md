# plans ディレクトリについて

## このディレクトリの役割

Issue ごとの実装計画を置く場所。
設計ドキュメント（`docs/design/`）が「何を作るか」を定義するのに対し、ここは「どう作るか」を記録する。

## ファイル命名

`<issue番号>-<概要>.md`（例: `003-セッション管理.md`）

## 実装計画に書くこと

- 実装の順序（何から作るか）
- 技術的な判断とその理由
- ブランチ内での作業中に決まったこと

## 書かないこと

- スコープと完了条件 → GitHub Issue に書く
- 機能の仕様 → `docs/design/` に書く
- プロジェクト全体の規約 → `CLAUDE.md` に書く
