# plans ディレクトリについて

## このディレクトリの役割

Issue ごとの実装計画を置く場所。
設計ドキュメント（`docs/design/`）が「何を作るか」を定義するのに対し、ここは「どう作るか」を記録する。

## ファイル一覧

- `<issue番号>-<概要>.md` — Issue ごとの実装計画（例: `003-セッション管理.md`）
- `todo.md` — まだ Issue にしていないが、議論の過程で生まれた対応が必要なことのメモ

## 実装計画の構成

### チェックリスト

大きな作業単位をチェックリストとして並べる。各項目は方針セクションの対応する見出しにリンクする。

```md
## チェックリスト

- [ ] [作業項目A](#作業項目a)
- [ ] [作業項目B](#作業項目b)
```

### 未決

作業中に出てきた「決める必要があるが今は決めない」ことを置く場所。チェックリスト（作業項目）とは別。決まったら方針に移す。

### 方針

チェックリストの各項目について、方針と実装方法を書く。作業に取りかかる前に議論して決める。

- **方針** — なぜそうするか、何を目指すか
- **実装方法** — どうやるか、具体的な手段

```md
## 方針

### 作業項目A

#### 方針

- 判断内容や理由

#### 実装方法

##### トピック名

- 具体的な実装手段
```

## 実装計画に書くこと

- 実装の順序（何から作るか）
- 技術的な判断とその理由
- ブランチ内での作業中に決まったこと

## 書かないこと

- スコープと完了条件 → GitHub Issue に書く
- 機能の仕様 → `docs/design/` に書く
- プロジェクト全体の規約 → `CLAUDE.md` に書く
